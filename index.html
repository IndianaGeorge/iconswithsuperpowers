Icons with Superpowers
Consistent default size between icons
  • SVG requires an explicit size
  • Different icon designs require different sizes
  • Icons used along text need a size relative to text
  • Align with text baseline during design
  • Crop the icon vertically within text bounds (icon cannot go above or below text)
  • Measure the offset as distance from baseline to bottom edge in SVG units, divided by the total viewbox height
  • Find the upper and lower edges of the icon in SVG units
  • Position icon as relative, vertically aligned to baseline, aplying the offset as top and a height of 1 em

Default color with support for fixed colors
  • How do we keep some colors untouched?
  • How can we customize through CSS?
  • Take a ClassName prop and append it to the class names
  • Take an optional color prop, default to currentColor (text color)
  • Apply color prop to paths, except where a specific color is meant by design
Icons with Superpowers
Solid and hollow variations
  • Which paths can become solid?
  • Need to address strokes and fills separately
  • How can we customize through CSS?
  • Define filled and stroke colors in objects reading from color prop, default to currrentColor, attribute-spread where necessary
  • Take separate props fillClassName and strokeClassName
  • Apply strokeClassName to every path
  • Apply fillClassName only to paths that can be filled

Disadvantages
  • No real font metrics in CSS, can only guess from reference fonts
  • 1em is not the actual height of a font, so we have to guess again
  • Enlarged inline icons will take full height space (ascender+descender), growing the line
  • Not friendly with atlasing
